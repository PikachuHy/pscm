# pscm GC Design

**Date**: 2026-05-01
**Status**: draft

## Goal

Add a mark-sweep garbage collector to pscm with conservative stack scanning. Prioritize readability and educational clarity. Reserve extension points for future generational / concurrent GC.

Non-goals: numeric tower changes, threading, moving GC.

## 1. Object Model

All GC-managed objects share a 16-byte header embedded before object data:


Fields:
- `mark` ‚Äî set during mark phase, cleared during sweep
- `age` ‚Äî reserved for generational GC (currently 0/1)
- `type_tag` ‚Äî discriminates object type; determines the trace function
- `size` ‚Äî total block size including header, in bytes
- `next_free` ‚Äî free-list pointer; only valid when block is unmarked (reuses object space)

### Tracked types and their pointer fields

| type_tag | object | pointer fields traced by GC |
|----------|--------|-----------------------------|
| `GC_SCM` | `SCM` | `value`, `source_loc` |
| `GC_LIST` | `SCM_List` | `data`, `next` |
| `GC_PROC` | `SCM_Procedure` | `name`, `args`, `body`, `env` |
| `GC_FUNC` | `SCM_Function` | `name` |
| `GC_CONT` | `SCM_Continuation` | `dst`, `arg`, `wind_chain`, `saved_module`, `stack_src` |
| `GC_MODULE` | `SCM_Module` | `obarray`, `uses`, `binder`, `eval_closure`, `transformer`, `name`, `public_interface`, `exports` |
| `GC_PORT` | `SCM_Port` | `string_data`, `output_buffer`, `soft_procedures` |
| `GC_VECTOR` | `SCM_Vector` | `data[]` (each element) |
| `GC_HASH` | `SCM_HashTable` | `buckets[]` (each entry) |
| `GC_ENV` | `SCM_Environment` | `dummy.next`, `dummy.data`, `parent` |
| `GC_STRING` | `SCM_String` | none (char data is not an SCM pointer) |
| `GC_SYMBOL` | `SCM_Symbol` | none |
| `GC_NUMBER` | SCM cells with inline value | none |

Each type_tag maps to a `TraceFn`:

```cpp
using TraceFn = void (*)(GCBlock *block, MarkStack *stack);
```

Example:

```cpp
void trace_scm(GCBlock *b, MarkStack *s) {
    SCM *scm = (SCM *)(b + 1);
    if (scm->value)  push_if_unmarked(s, to_block(scm->value));
    if (scm->source_loc) push_if_unmarked(s, to_block(scm->source_loc));
}
```

### Allocation API

Replace `new SCM()` / `new SCM_List()` with:

```cpp
void *gc_alloc(TypeTag tag, size_t size);
```

`wrap<T>()` templates call `gc_alloc` internally.

## 2. Root Set & Conservative Stack Scan

### Roots

**Explicit roots** (registered via addresses): `g_env`, `g_wind_chain`, `g_root_module`, `g_current_module`, `g_current_eval_context`, `g_eval_stack`, `g_catch_stack`, `g_error_key`, `g_eof_object`, `g_module_registry`.

**C stack**: scan from `cont_base` to current stack top, one `long` at a time.

**Registers**: snapshot via `setjmp` into a `jmp_buf`, scan each slot.

### Conservative scan algorithm

```cpp
void scan_conservative_roots(MarkStack *s) {
    // 1. explicit roots
    for each registered root: mark_if_scm_pointer(*root, s);

    // 2. C stack
    long stack_top;
    long *lo = (long *)&stack_top;
    long *hi = (long *)cont_base;
    if (lo > hi) swap(lo, hi); // adapt to stack growth direction
    for (long *p = lo; p < hi; p++)
        mark_if_valid_block((void *)*p, s);

    // 3. registers
    jmp_buf regs;
    setjmp(regs);
    for (size_t i = 0; i < sizeof(regs) / sizeof(long); i++)
        mark_if_valid_block((void *)regs[i], s);
}
```

`mark_if_valid_block(p, s)` does:
1. Is `p` within GC heap range (`heap_start` ‚â§ p < `heap_end`)?
2. Is `p` aligned to a block start? (check header magic or block-start bitmap)
3. If yes, set `mark` bit and push to `MarkStack`.

False positives (non-pointer integer that looks like a heap address) only keep extra objects alive ‚Äî no correctness issue. False negatives are avoided by the heap-range and alignment checks.

### What does NOT need changes

- Local `SCM *` variables in C/C++ ‚Äî automatically covered by conservative stack scan.
- C API callers using `pscm_api.h` ‚Äî no changes required.

## 3. Mark-Sweep Cycle

### Mark phase

Uses an explicit mark stack (`std::vector<GCBlock*>` or fixed-capacity array) to avoid C stack overflow from deep object graphs:

```cpp
void mark_phase() {
    MarkStack stack;
    scan_conservative_roots(&stack);
    while (!stack.empty()) {
        GCBlock *b = stack.pop();
        trace_fns[b->type_tag](b, &stack);
    }
}
```

`push_if_unmarked`: if block is not yet marked, set its `mark` bit and push to stack.

### Sweep phase

Linear scan over all heap segments:

```cpp
void sweep_phase() {
    for each segment in heap_segments {
        for (GCBlock *b = segment_start; b < segment_end; b = next(b)) {
            if (b->mark) {
                b->mark = 0;     // clear for next cycle
            } else {
                free_to_list(b);
            }
        }
    }
    // truly empty segments ‚Üí munmap
}
```

### GC trigger

```cpp
void *gc_alloc(TypeTag tag, size_t size) {
    if (allocated_since_gc > threshold || free_list_empty(size)) {
        run_gc();
        allocated_since_gc = 0;
        if (free_list_empty(size))
            grow_heap();
    }
    void *b = take_from_free_list(size);
    allocated_since_gc += header_size + size;
    return b;
}
```

`threshold` = 1.5√ó live bytes from last GC, with a floor of initial heap / 4 (~2MB).

### Invariants

- Objects are never moved ‚Äî all SCM pointers remain valid across GC.
- `mark` bits are all zero outside a GC cycle.
- `next_free` is only read on blocks that are in the free list (unmarked && freed).

## 4. Free List

Six size-segregated buckets:

| bucket | size | typical objects |
|--------|------|-----------------|
| 0 | ‚â§ 16B | tiny SCM cells (inline values) |
| 1 | 17‚Äì32B | SCM with source_loc |
| 2 | 33‚Äì64B | SCM_List, SCM_Symbol, SCM_Function |
| 3 | 65‚Äì128B | SCM_Procedure, SCM_Port, SCM_Continuation |
| 4 | 129‚Äì512B | SCM_Module, small SCM_Vector, SCM_String |
| 5 | 513B+ | large objects ‚Äî separate mmap, returned via munmap |

### Allocation

```cpp
void *take_from_free_list(size_t size) {
    int bucket = size_to_bucket(size);
    if (free_lists[bucket]) {
        GCBlock *b = free_lists[bucket];
        free_lists[bucket] = b->next_free;
        b->next_free = nullptr;
        return b;
    }
    return bump_allocate(size, bucket);
}
```

### Return (LIFO, cache-friendly)

```cpp
void free_to_list(GCBlock *b) {
    int bucket = size_to_bucket(b->size);
    b->next_free = free_lists[bucket];
    free_lists[bucket] = b;
}
```

Buckets 0‚Äì3: fixed-size blocks per bucket, no external fragmentation.
Bucket 4: best-fit with splitting.
Bucket 5: per-object mmap/munmap.

### Heap expansion

Initial heap: 8MB via `mmap` + `MAP_ANON`. Additional segments allocated on demand, linked as a singly-linked list. GC sweep walks all segments.

## 5. Code Integration

### Files to add

- `src/c/gc.h` ‚Äî GC header, GCBlock, type tags, public API
- `src/c/gc.cc` ‚Äî allocator, mark, sweep, free list, heap management

### Files to modify

- `pscm.h` ‚Äî all `wrap<T>()` factories switch from `new` to `gc_alloc`; type enum augmented with GC type_tag mapping
- `init.cc` ‚Äî call `gc_init()` at start
- `eval.cc` ‚Äî `cont_base` registration aligned with GC stack scan range
- All payload constructors (`SCM_List`, `SCM_Procedure`, etc.) ‚Äî replace `new` with `gc_alloc`

### What stays on malloc/free

- `SCM_String::data`, `SCM_Symbol::name` ‚Äî char buffers, no SCM pointers, released by ~SCM_String / ~SCM_Symbol destructors
- `SCM_Port::string_data`, `SCM_Port::output_buffer` ‚Äî char buffers, freed on port close
- `SCM_Continuation::stack_data` ‚Äî raw C stack bytes, freed by `scm_dynthrow` or continuation destruction
- `SCM_Smob::data1/2/3` ‚Äî user data, managed by smob free callback
- `SCM_Port::file` ‚Äî FILE*, managed by port close

### Initialization

`gc_init()` called at the top of `init_scm()`:
- mmap initial 8MB heap segment
- register explicit root addresses
- compute initial `threshold`

## 6. Testing

### Scheme-level correctness tests (`test/gc/`)

1. **Basic reachability** ‚Äî GC between define and access must not reclaim live objects
2. **Cyclic references** ‚Äî mark phase must terminate and not collect cycles reachable from roots
3. **Closure capture** ‚Äî GC must trace procedure environments
4. **Large allocation** ‚Äî triggers heap expansion
5. **Repeated GC** ‚Äî mark bits correctly reset across cycles (100 GCs while object is live)
6. **dynamic-wind + GC** ‚Äî wind chain not reclaimed while active
7. **Continuation + GC** ‚Äî saved continuation stack data and wind chain survive GC

### C++ unit tests (`test/unit/`)

8. Empty heap GC ‚Äî no crash
9. Free-list exhaustion triggers heap growth
10. Full-free-then-GC ‚Äî no use-after-free
11. Conservative scan false-positive determinism test

### Stress tests

12. Allocate-and-drop loop (10k iterations, triggers hundreds of GCs)
13. Full R4RS/R5RS test suite regression ‚Äî no existing tests break

### Build integration

```bash
ninja -C out/build/pscm-cmake check-gc  # new lit test suite
```

## 7. Extension Points (for future generational GC)

- `age` field in GC header ‚Äî currently unused, reserved for generational promotion
- `TraceFn` function table ‚Äî can add write-barrier hooks without changing mark/sweep core
- `gc_alloc` accepts type_tag ‚Äî can route young objects to a nursery bump-pointer allocator later
- `free_to_list` / `take_from_free_list` ‚Äî can be swapped with segment-level reclamation
